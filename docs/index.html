node_a

The node_a script is a core component of the rt_assignment2 package. This node handles essential operations that form the backbone of the assignment’s functionality. It interacts with the ROS action server to send movement goals and cancel them when required. Additionally, it subscribes to odometry data to update the robot’s position and velocity.

Subscriber:

  • /node_a/pose: Subscribes to the pose.

scripts.node_a.main()[source]

Initialize the ROS node and start handling goal commands.

This is the main entry point of the script. It initializes the ROS node, sets up the necessary components, and starts the process of handling user commands for setting and cancelling goals.

scripts.node_a.process_goal_commands(pub, client, goal_cancelled)[source]

Handle goal commands based on user input, either setting new goals or cancelling existing ones.

This function continuously listens for user commands to either set a new goal or cancel the current goal. It subscribes to the /odom topic to receive the robot’s current odometry information, which it uses to update the robot’s position and velocity.

Parameters:

  • pub (rospy.Publisher) – The ROS publisher for position and velocity.

  • client (actionlib.SimpleActionClient) – The ROS action client for the ‘reaching_goal’ action.

  • goal_cancelled (bool) – A flag indicating if the current goal is cancelled.

scripts.node_a.publish_position_velocity(msg, pub)[source]

Publish position and velocity based on the received odometry message.

This function extracts the robot’s current position and velocity from the odometry message and publishes it using the provided publisher.

Parameters:

  • msg (nav_msgs.msg.Odometry) – The odometry message containing position and velocity data.

  • pub (rospy.Publisher) – The ROS publisher for position and velocity.

scripts.node_a.ros_components_init()[source]

Initialize ROS components including the publisher and action client.

This function sets up the necessary ROS components for the application: - A publisher for publishing the robot’s position and velocity. - An action client for sending goals to the ‘reaching_goal’ action server. - A flag to indicate whether the current goal has been cancelled.

Returns:

A tuple containing:

  • pub (rospy.Publisher): The ROS publisher for position and velocity.

  • client (actionlib.SimpleActionClient): The ROS action client for the ‘reaching_goal’ action.

  • goal_cancelled (bool): A flag indicating if the current goal is cancelled.

Return type:

tuple

node_b

The node_b script provides a service that allows for the retrieval of the last desired x and y positions. This functionality is essential for tracking and managing the robot’s movement goals. By storing the desired positions as ROS parameters, this node ensures that other components of the system can access the last target positions whenever needed.

Module author: Seyed Alireza Mortazavi

This code implements Node B.

Subscriber:

  • /node_b/pose: Subscribes to the pose topic.

scripts.node_b.handle_input_request(request)[source]

Handle the service callback to respond with the last desired target positions.

This callback function is triggered whenever a service request is made to the ‘input’ service. It retrieves the last desired x and y target positions from the ROS parameter server and constructs a response object containing these positions.

Parameters:

request (assignment_2_2023.srv.InputRequest) – The request object from the service call, which is part of the ‘Input’ service type. The request is not used in this function as the response is generated based on stored parameters.

Returns:

The response object containing the last desired x and y positions.

Return type:

assignment_2_2023.srv.InputResponse

scripts.node_b.ros_node_init()[source]

Initialize the ROS node and provide the ‘input’ service.

This function sets up the ROS infrastructure for handling service requests regarding the last desired target positions. Specifically, it initializes a ROS node named ‘last_target_service’ and advertises a service named ‘input’ that uses the custom service type ‘Input’.

The function also logs an informational message indicating that the node has been initialized.

Side Effects:

  • Initializes a ROS node named ‘last_target_service’.

  • Sets up a ROS service named ‘input’ that calls the handle_input_request callback.

scripts.node_b.run_ros_node()[source]

Keep the ROS node running.

This function enters a loop, keeping the ROS node active and responsive to incoming service calls. It effectively keeps the program alive until it is manually terminated.

Side Effects:

  • Keeps the ROS node running and processing callbacks.

node_c

The node_c script logs real-time data of the robot’s position, velocity, and other relevant metrics. This node is crucial for monitoring the robot’s current state and performance, ensuring that the system has access to up-to-date information at all times. It subscribes to position and velocity topics, calculates distances and average velocities, and provides this information through a ROS service.

Module author: Seyed Alireza Mortazavi

Subscribers:

  • /pos_vel: Subscribes to the position and velocity information.

Services:

  • info_service: Provides real-time info about distance and velocity.

scripts.node_c.calculate_distance_and_average_velocity(msg)[source]

Callback function for the subscriber to calculate distance and average velocity.

This function is triggered upon receiving a message on the ‘/pos_vel’ topic. It calculates the Euclidean distance between the desired target positions and the actual positions received in the message. Additionally, it calculates the average velocity over a specified window size. These values are updated on the ROS parameter server.

Parameters:

msg (assignment_2_2023.msg.Vel) – The received message containing position and velocity data.

Side Effects:

  • Updates the ‘/distance’ and ‘/average_vel_x’ parameters on the ROS parameter server.

  • Logs the calculated distance and average velocity information.

scripts.node_c.handle_info_request(_)[source]

Handle service requests to provide distance and average velocity.

This function is triggered when a service request is made to the ‘info_service’ service. It retrieves the previously calculated distance and average velocity from the ROS parameter server and returns these values in the service response.

Returns:

The response object containing the distance and average velocity.

Return type:

assignment_2_2023.srv.Ave_pos_velResponse

scripts.node_c.ros_node_init()[source]

Initialize the ROS node, service, and subscriber.

This function initializes a ROS node named ‘info_service’, which provides a service named ‘info_service’ using the custom service type ‘Ave_pos_vel’. It also subscribes to the ‘/pos_vel’ topic to receive messages containing position and velocity data.

Side Effects:

  • Initializes a ROS node.

  • Sets up a ROS service named ‘info_service’.

  • Sets up a ROS subscriber to the ‘/pos_vel’ topic.

scripts.node_c.run_ros_node()[source]

Keep the ROS node running.

This function keeps the ROS node active and responsive to incoming service calls and subscriber messages. It effectively maintains the node’s operational state until it is manually terminated.

Side Effects:

  • Keeps the ROS node running and processing callbacks.
